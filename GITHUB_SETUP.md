# 📚 Полная инструкция: загрузка на GitHub Pages

## Часть 1️⃣ — Создание репозитория

### Шаг 1.1 — Новый репозиторий на GitHub
1. Откройте https://github.com → войдите в аккаунт
2. Нажмите **"+"** (верхний правый угол) → **"New repository"**
3. Заполните:
   - **Repository name:** `music-catalog` (или своё имя)
   - **Description:** 🎵 Каталог музыки с тегами — настроение, стиль, назначение
   - **Public** (обязательно для Pages!)
   - **Add a README file** — оставьте пусто, добавим свой

4. Нажмите **"Create repository"**

### Шаг 1.2 — Загрузка файлов (первый коммит)

На странице репозитория:
1. Нажмите **"Add file"** → **"Upload files"**
2. Перетащите в окно три файла:
   - `index.html` (само приложение)
   - `catalog.json` (база с вашей музыкой)
   - `README.md` (описание проекта)

3. Внизу заполните:
   - **Commit message:** `Initial commit: add Music Catalog app`
   - **Extended description:** `
Add Music Catalog application with:
- Album and track management
- Tags: mood, purpose, style, instruments, national color, tempo
- Spotify links and audio file support
- GitHub integration for auto-save
- Export to CSV and JSON
- Full offline support with localStorage
   `

4. Выберите **"Commit directly to the main branch"**
5. Нажмите **"Commit changes"**

---

## Часть 2️⃣ — Включение GitHub Pages

### Шаг 2.1 — Настройка Pages

В репозитории:
1. Перейдите в **Settings** (шестерёнка, правый угол)
2. В левом меню найдите **"Pages"**
3. Под **"Source"** выберите:
   - **Deploy from a branch**
   - Branch: **main** (или master, в зависимости от версии)
   - Folder: **/(root)**

4. Нажмите **Save**

⏳ Подождите 1–2 минуты. Вверху появится зелёная галочка и ссылка:

```
✅ Your site is live at https://ваш-ник.github.io/music-catalog/
```

**Всё!** Приложение доступно по этой ссылке.

---

## Часть 3️⃣ — Создание GitHub Token (для редактирования через приложение)

Это нужно только, если хотите **автоматически сохранять изменения на GitHub** прямо из приложения.

### Шаг 3.1 — Создание токена

1. Откройте https://github.com/settings/tokens
2. Нажмите **"Generate new token"** → **"Generate new token (classic)"**
3. Заполните:
   - **Token name:** `music-catalog-token`
   - **Expiration:** 90 days (или No expiration)
   - **Select scopes:** поставьте галочку на **repo** (всё под ней будет отмечено)

4. Нажмите **"Generate token"** внизу

⚠️ **СКОПИРУЙТЕ ТОКЕН СРАЗУ!** Он больше не будет виден.
Сохраните в безопасном месте (Password Manager, заметки).

### Шаг 3.2 — Подключение в приложении

1. Откройте свой каталог: https://ваш-ник.github.io/music-catalog/
2. Нажмите кнопку **⚙ GitHub** (верхний правый угол)
3. Заполните:
   - **GitHub Token:** вставьте токен из шага 3.1
   - **Repository:** `ваш-ник/music-catalog` (ваш username/имя репо)
   - **Path to file:** `catalog.json` (оставьте как есть)

4. Нажмите **"Проверить соединение"** — должна быть зелёная галочка ✅
5. Нажмите **"Сохранить"**

Готово! Теперь каждый раз, когда вы редактируете:
- Трек → коммит: `Update track: "название" in "альбом"`
- Альбом → коммит: `Add album: "название" by Исполнитель`
- Обложку → коммит: `Update cover: "альбом"`

## Часть 4️⃣ — Дальнейшие коммиты (ручные)

Если вы хотите добавлять треки / альбомы на GitHub вручную (через веб-интерфейс):

### Коммит при добавлении трека вручную

1. В GitHub → нажмите на `catalog.json`
2. Нажмите **✏️ Edit this file**
3. Отредактируйте JSON (добавьте новый трек в массив tracks)
4. Внизу:
   - **Commit message:** `Add track: "название трека" to "название альбома"`
   - **Extended description:**
   ```
   Artist: Исполнитель
   Album: Название альбома
   Track: Название трека
   Duration: 3:45
   ```
5. Нажмите **"Commit changes"**

### Коммит при обновлении альбома

**Commit message:**
```
Update album metadata: "название альбома"
```

**Extended description:**
```
Updated:
- Spotify URL: https://...
- Cover image: [новая обложка]
- Release date: YYYY-MM-DD
```

### Коммит при добавлении обложки

**Commit message:**
```
Update cover: "название альбома"
```

## Часть 5️⃣ — Просмотр истории изменений

Все коммиты видны в **GitHub → Commits**:

1. В репозитории нажмите на кнопку **📊 Commits** (вверху, слева)
2. Видите список всех коммитов с описанием и датой
3. Кликните на коммит → видите, что именно изменилось

**Пример истории:**
```
Initial commit: add Music Catalog app — 25 апр 12:35
Update album metadata: "Slow Down Soundscapes" — 25 апр 13:10
Add track: "New Morning" to "Northern Landscapes" — 25 апр 13:45
Update cover: "Winter Ambient" — 25 апр 14:02
Add album: "Jazz Evening" by Focus State Way — 25 апр 14:30
```

## Часть 6️⃣ — Поделиться каталогом

### Отправить ссылку на просмотр

Дайте знакомым эту ссылку:
```
https://ваш-ник.github.io/music-catalog/
```

Они смогут:
- ✅ Просматривать альбомы и треки
- ✅ Слушать на Spotify (если есть ссылки)
- ✅ Искать по тегам
- ❌ Редактировать (только вы)

### Кто может редактировать?

- **Вы** — есть токен GitHub
- **Никто другой** — даже если найдёт токен, он не может его украсть (хранится только в вашем браузере)

### Если хотите дать доступ другому редактору

1. В репозитории → **Settings** → **Collaborators**
2. Нажмите **"Add people"**
3. Напишите username того, кому дать доступ
4. Он получит приглашение и сможет редактировать прямо в GitHub

## Часть 7️⃣ — Часто задаваемые вопросы

### ❓ GitHub Pages показывает 404
**Решение:**
- Проверьте Settings → Pages
- Убедитесь, что branch = main и folder = /(root)
- Подождите 2–3 минуты
- Очистите кэш браузера (Ctrl+Shift+Del)

### ❓ Приложение загружается, но каталог пуст
**Решение:**
- Проверьте, что `catalog.json` загружен в репозиторий
- Откройте консоль браузера (F12) — посмотрите ошибки
- Попробуйте перезагрузить страницу (Ctrl+F5)

### ❓ Коммиты не создаются автоматически
**Решение:**
- Проверьте, что токен заполнен в ⚙ GitHub
- Кликните "Проверить соединение" — должна быть ✅
- Если ❌ — создайте новый токен (старый может быть просрочен)

### ❓ Ошибка 401 при сохранении на GitHub
**Решение:**
- Токен истёк (если выбрали 90 дней) → создайте новый
- Имя репо неправильно → проверьте `username/music-catalog`
- Токен потерял права → создайте с галочкой **repo**

### ❓ Хочу сделать каталог приватным
**Решение:**
- Settings → **Private** (требует GitHub Pro)
- Или создайте второй приватный репозиторий для себя

### ❓ Как скачать копию каталога?
**Решение:**
- В приложении нажмите **↓ JSON** — скачается `catalog.json`
- Или **↓ CSV** — для Excel / Google Sheets

## 📊 Идеальная структура репозитория

После всех шагов у вас должно быть:

```
music-catalog/
├── index.html          ← приложение
├── catalog.json        ← база данных
├── README.md           ← описание (этот файл)
└── .gitignore          ← (опционально, исключить файлы)
```

**Больше ничего не нужно!** GitHub Pages сам развернёт `index.html`.
